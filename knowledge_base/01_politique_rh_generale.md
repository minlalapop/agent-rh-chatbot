# Politique RH Générale

## 1. Objet

Ce document définit les règles générales de gestion des ressources humaines au sein de l’entreprise.  
Il encadre les interactions entre les employés, les managers et le service RH, ainsi que les procédures administratives standard.

---

## 2. Rôle du service RH

Le service des Ressources Humaines est chargé de :

- La gestion administrative des salariés (contrats, paie, absences, documents)
- L’accompagnement des employés dans leurs démarches administratives
- La conformité aux obligations légales et conventionnelles
- La gestion des données personnelles
- Le support aux managers

Le service RH intervient principalement sur :
- contrat de travail
- rémunération
- congés et absences
- documents administratifs

---

## 3. Canaux de contact

Les employés peuvent contacter le service RH via :

- Portail RH interne (SIRH) — canal principal
- Email RH : rh@entreprise.com
- Système de ticketing interne
- Entretien RH (sur rendez-vous)

Toute demande doit être claire et contenir les informations nécessaires.

---

## 4. Outils internes

Les outils utilisés sont :

- **SIRH (portail salarié)** : gestion des congés, accès documents, informations personnelles
- **Coffre-fort numérique** : stockage des contrats et documents RH
- **Système de ticketing RH** : suivi des demandes complexes
- **Annuaire interne** : identification du manager et des équipes

---

## 5. Délais de traitement

| Type de demande | Délai moyen |
|----------------|------------|
| Accès document | immédiat à 24h |
| Modification données personnelles | 2 à 5 jours ouvrés |
| Attestation / document RH | 3 à 7 jours ouvrés |
| Incident paie | 2 à 5 jours ouvrés |
| Modification contractuelle | 5 à 15 jours ouvrés |

Les délais peuvent varier selon la complexité.

---

## 6. Confidentialité

- Les données RH sont strictement confidentielles
- Un employé ne peut accéder qu’à ses propres informations
- Les managers ont un accès limité à leur équipe
- Le service RH a un accès complet aux données administratives

Toute demande impliquant des données sensibles nécessite validation RH.

---

## 7. Contexte utilisateur

Les réponses RH dépendent du profil :

- type de contrat
- ancienneté
- département
- rôle (employee, manager, hr)

Certaines règles ou procédures peuvent varier selon ces éléments.

---

## 8. Validation hiérarchique

Certaines actions nécessitent validation :

- Congés → validation manager
- Modifications contractuelles → validation RH + manager
- Demandes exceptionnelles → validation RH

Le manager (N+1) est le premier niveau de validation.

---

## 9. Cycle de traitement des demandes

Une demande RH suit généralement :

1. Soumission par l’employé  
2. Vérification des informations  
3. Traitement par RH ou manager  
4. Validation si nécessaire  
5. Retour à l’employé  

Une demande incomplète peut être suspendue.

---

## 10. Limites du service RH

Le service RH ne traite pas :

- les demandes non administratives
- les décisions purement opérationnelles (relevant du manager)
- les demandes incomplètes

---

## 11. Résumé opérationnel

- Le service RH gère les demandes administratives
- Le SIRH est l’outil principal
- Certains processus nécessitent validation
- Les délais varient selon la complexité
- Les réponses dépendent du profil utilisateur
